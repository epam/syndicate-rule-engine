# SRE Event-Driven SQS Loader

Script to emulate Event-Driven events (AWS / MAESTRO) and publish them to SQS. Tenants and events are configured via YAML.

## Installation

```bash
pip install -r scripts/ed_high_loader/requirements.txt
```

AWS credentials (env, `~/.aws/credentials`, or IAM role) are required to send messages to SQS.

## Configuration

1. Copy the example config:
   ```bash
   cp scripts/ed_high_loader/config.example.yaml scripts/ed_high_loader/config.yaml
   ```
2. In `config.yaml` specify:
   - **sqs.queue_url**, **sqs.region**
   - **aws_vendor.AWS** — `account_ids`, `regions` (optional)
   - **maestro_vendor** — one key per cloud (GOOGLE, AZURE, AWS, etc.): `tenant_names`, `regions`
3. Events are defined in **events.yml** (default: `scripts/ed_high_loader/events.yml`): one key per cloud, list of dicts `event_source → [event_name]`. This file is generated by `ed_events_sync`.

**Behavior:** For each (tenant, region) pair, events are split into chunks of 1–5 events per message (random). Each SQS message contains one chunk of events.

### AWS

- `aws_vendor.AWS`: `account_ids`, `regions`. Events are taken from `events.AWS` in `events.yml`. Messages include `detail-type: "AWS API Call via CloudTrail"`.

### MAESTRO

- `maestro_vendor.<cloud>`: `tenant_names`, `regions`. Events are taken from `events.<cloud>` (same key, e.g. GOOGLE, AZURE, AWS).

Other options:
- **message_count** — how many times to send the payload set (default: 1).
- **delay_seconds** — delay in seconds between sends (0 = no delay).

## Usage

Run from project root:

```bash
# Send to SQS (config: scripts/ed_high_loader/config.yaml, events: scripts/ed_high_loader/events.yml)
python -m scripts.ed_high_loader

# Different config
python -m scripts.ed_high_loader -c scripts/ed_high_loader/config.yaml

# Different events file
python -m scripts.ed_high_loader -e events.custom.yml

# Dry run: only print payloads, do not send
python -m scripts.ed_high_loader --dry-run

# Parallel send (N threads)
python -m scripts.ed_high_loader -w 4
```

### Arguments

| Option | Description | Default |
|--------|-------------|---------|
| `-c`, `--config` | Path to config | `scripts/ed_high_loader/config.yaml` |
| `-e`, `--events` | Path to events file | `scripts/ed_high_loader/events.yml` |
| `-n`, `--dry-run` | Do not send to SQS, only print payloads | — |
| `-w`, `--workers` | Number of worker threads for sending | 1 |

## Event format

Payload wrapper: `version`, `vendor`, `events[]`. For AWS: CloudTrail-style `detail` with `detail-type: "AWS API Call via CloudTrail"`. For MAESTRO: `tenantName`, `regionName`, `eventMetadata`.
